4.7 Importing AADL Models into OSATE
AADL models may have been created outside OSATE.  For example, we have provided a number of AADL model examples in textual form that can be downloaded from http://www.aadl.info/AADLModels.html.  Such files may be made available as a collection of files, as an OSATE or Eclipse project in a zip file. First, we describe how AADL models can be imported as files, then we describe how an Eclipse or OSATE project can be imported.
AADL models can be brought into an OSATE workspace as files:
    • You can drag the file from the file system into the OSATE Workspace.
    • Files can be imported into a project in the workspace.  This is done by selecting “File > Import…” then selecting “from File System” in the “Import” dialog box.  The files being imported are copied from the file system location into the workspace location. 
AADL model files may have been made available to you as an OSATE project in a zip file.  In this case, unzip the file into an OSATE workspace from outside Eclipse, then from within Eclipse/OSATE invoke “Import > Existing Project into Workspace” and point to the folder in the OSATE workspace folder that represents the project you just unzipped.
